# containerSecurity
Securing Containers
This repository is a hands-on guide to securing containers.

Purposefully, start with insecure defaults, then progressively harden the container so each concept builds naturally on the previous one.

What to Learn
---------------

- How to run containers as a non-root user
- How to harden containers at runtime
- Why running containers as root is dangerous
- Why .dockerignore is critical
- Why distroless images are more secure
